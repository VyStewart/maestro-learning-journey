## Lesson 15: Demonstrating the working app
### Date: Jan 18 - 2026

**Core Focus**
- Proving that a complete application works by validating core logic with assert statements.

**Key Concepts**
- Logic should be isolated in testable functions.
- Interactive input is difficult to test automatically.
- assert statements demonstrate correctness mechanically.

**Demonstration Pattern**
- Write core logic as a pure function.
- Build the app around that logic.
- Use assert to verify expected behavior.
- Let passing tests confirm correctness.

**Why This Matters**
- Prevents silent logic bugs.
- Makes refactoring safer.
- Builds confidence based on proof, not assumption.
