### lesson 01: Welcome to algorthms 

1. Core idea
 
- Algorithm = a clear, step‑by‑step method to get something done or solve a problem.
    - Example: a cooking recipe with numbered steps.

2. Algorithms vs data structures
 
- Data structure = how we store data
    Examples: list, stack, queue

- Algorithm = how we act on that data
    Example: “while the queue is not empty, take the next person and serve them”

3. The algorithms unit is about:
 
    - Turning ideas into ordered steps
    - Using those steps with lists/stacks/queues to actually do something with the data

4. What the algorithms unit will cover (at a high level)
 
    - Turning messy problems into clean step‑by‑step plans
    - Counting how much work an algorithm does (for small vs large inputs)
    - Building intuition for growth and hearing the term Big‑O (a way to talk about how fast/slow an algorithm grows — no heavy math yet)
    - Comparing different solutions and choosing better ones
    - Thinking about edge cases (empty list, one item, huge input) so your code doesn’t break

 5. Why algorithms matter: 
 
- Learning algorithms will help you:
 
    - Write faster, more efficient Python code
    - Plan your approach before coding
    - Debug more easily because your steps are clear
    - Prepare for interviews and real-world problems
